Ruby Basics: Classes

Classes in Ruby

2/26/2016

Today I'll be talking a bit about ruby classes and class objects. If you read my post on Arrays and Hashes, you know that classes are basically specifications for a whole group of Ruby objects. Today, I'll be using video games as an example of classes.

Let's start by defining a video game. Video games have a rating, a genre, a title, and a number of players. So if we wanted to set up a video game class in Ruby, it would need all of those different characteristics. To make that happen, we can set up

instance variables
using the "@" sign. Instance variables are like the different aspects of a class-they are part of every object in that class, but they can change between each new instance within that class. Here's an example of me setting up the
VideoGames
class and setting up instance variables for video games:

class VideoGames

def initialize(title, genre, rating, players)

@title = title

@genre = genre

@rating = rating

@players = players

end

end

In that example, I made VideoGames a class. I also defined a special method called
initialize
. That way when a new game is made, I can store its title, genre, rating, and number of players in instance variables as soon as I tell ruby that the game is part of the VideoGame class. Here's an example:

new_game("Little Big Planet", "Platformer", "E", "1-4") = VideoGame.new

That's all! I have set up a VideoGame called new_game that has a title called "Little Big Planet," it belongs to the "platformer" genre, its rating is "E", and it can be played with 1 to 4 players.

Now that our new VideoGame class is storing information, we have to find a way to do something with all of that info. This is where

instance methods
come in. Instance methods are actions that can be performed on a specific class, so if we want to write a program that takes the name, genre, rating, and number of players for a game we'd have to write it into the definition for that class. This is how we'd do that:

class VideoGames

def initialize(title, genre, rating, players)

@title = title

@genre = genre

@rating = rating

@players = players

end

def game_description

puts "Title: #{@title}"

puts "Genre: #{@genre}"

puts "Rating: #{@rating}"

puts "Players: #{@players}"

end

end

That's our original example, but with an instance method called
game_description
written in. Now, when we define our new_game as an object in the VideoGame class, we can also print out the game's description as well. Here's how:

new_game("Little Big Planet", "Platformer", "E", "1-4") = VideoGame.new

new_game.game_description

This is what will print out when we run that code:

Title: Little Big Planet

Genre: Platformer

Rating: E

Players: 1-4

When we called the instance method
game_description
on our
new_game
, it took the title, genre, rating, and title that we supplied it with and printed each bit of info out on individual lines.

Classes are tremendously useful when you have an entire category of objects that have the same kinds of characteristics. You can practically set up classes for anything-cars, computers, books, movies, you name it! By using classes, you can quickly and easily collect info about an object within that class and perform actions using that provided data.
