Ruby Basics

Containers

2/13/16

In the programming language Ruby there are many "things"--technically referred to as

objects
--that make up a program, and today we will be talking about a couple of objects, called
arrays
and
hashes
, that can
contain
, well, other objects. Because arrays and hashes can contain other objects they belong to a
class
, or group classification, of objects called
containers
. Let's take a look at how these containers hold other objects, and how Ruby allows us to utilize containers to arrange, organize, and modify their contents.

Arrays

Arrays are the first type of container we'll talk about. Arrays are lists of objects that are separated by commas, contained in brackets, and sorted in and

indexed numerical order
(I'll elaborate on that more later). An example array would look like this:
example_array = ["red", "blue", "green", "yellow"]
In that little block of code, the array's name is
example_array
and there are 4 string objects in the array:
"red", "blue", "green",
and
"yellow"
. These objects are sorted in numerical order starting at 0, and their place in the array is known as their
index
. Since 0 is the starting point for indexes, the first object in the array is
"red"
and its index is 0; therefore
"red"
has the
0th index
in the array. Thanks to this indexing system, we can refer to an individual object in an array by typing this in Ruby (with
n
being the index number you want to refer to):
array_name[n]
This makes it easy to store data in an array and access an individual object within that array whenever you need it!

It's also important to mention that you can perform

methods
--defined sequences of actions in Ruby--on all of the items in an array by typing the name of the array, followed by the method you would like to apply to it. Here's an basic example:
array_name.methodname
In the above instance, the array
array_name
is being modified by the fake method, called
methodname
. That is how you apply a method to every object in an array; whatever method goes after the period following the array name will perform its designated actions to each item in the array and return whatever it is meant to return. Using methods on entire arrays is a great way to mass-edit or organize lists of data and save yourself a lot of work!

Hashes

Hashes are the other container type we'll look at. Hashes are also lists of objects, but they are stored and sorted in pairs of

keys
and
values
, separated by commas, contained in curly braces, and each pair is sorted as a whole based on the order it was added into the array. While that may sound like quite a bit to take in at first, it's actually very similar to how we would look at an array. Here's an example of a hash:
color_hash = { "best" => "green" "second best" => "red" "third best" => "yellow" }
In the above example, the hash is named
color_hash
, the keys are
"best", "second best",
and
"third best"
, the values are
"green", "red",
and
"yellow"
, and the key-value pairs are
"best" | "green", "second best" | "red",
and
"third best" | "yellow"
. Let's break that down. The keys in a hash are like the index for an array; you can use each key to refer to its corresponding value in a hash. For example, just as
example_array[0]
returns an output of
"red"
from the array
example_array
, you can type
color_hash["best"]
and that will return
"green"
from the
color_hash
hash. When you look at them side by side like that, array indexes and hash keys are pretty identical! The only major difference is that array indexes can only be positive integers, whereas hash keys can be any type of Ruby object at all. Another important thing to note about keys and values is that there can only be one value for a key; basically all that means is that each key can only be used once in any given key-value pair in a hash. Think about it: if keys in a hash are like the indexes in an array, each key can only reference one value-just like how each index can only reference one object.

Just like arrays, hashes can also be modified using methods. You can actually use the exact same syntax! Here's an example of how you'd apply a method to a hash:

hash_name.methodname
Look familiar? That example demonstrates that hash methods are applied in the same way as array methods; list the name of the hash, followed by a period, then finally followed by the name of the method you'd like to apply.

Summary

In conclusion, both hashes and arrays are useful containers for Ruby; what youre probably thinking now is, "Why use one over the other?" I've found that arrays tend to be more useful the objects in the array will be changing often. For example, I'd prefer to use an array if I wanted to make a program that calculated the average of a list of numbers I put into it. Hashes are more useful when you know exactly what keys and values you're going to put into the list and you just want to be able to reference them easily. For instance, if I wanted to write a program that determined the measurement, in inches, of whatever size I enter in(from small to extra extra large), I'd make a hash that uses the entered size as the keys and the inch measurments as the corresponding values. I hope my take on arrays and hashes helped you out!
